Enterprise Architect

Thank you for purchasing Enterprise Architect 9.3 on CD

Enterprise Architect is a comprehensive, full lifecycle, analysis, design and construction tool. Enterprise Architect supports software development from early requirements gathering, through the analysis stages, design models, testing and finally maintenance and re-use. Enterprise Architect supports all 14 UML 2.4 diagram types, numerous software languages and a wealth of high-end features.

Model Driven Generation (MDG) Integration products tightly integrate Enterprise Architect, UML and related modeling standards with popular Integrated Development Environments (IDEs) such as Visual Studio, Eclipse and Teamcenter Systems Engineering. MDG Technology

Model Driven Generation (MDG) Technologies extend Enterprise Architect's core modeling capabilities to domain specific languages, UML profiles and frameworks such as Zachman Framework, TOGAF, DoDAF-MODAF, SysML, DDS and many more. MDG Technologies not only expand available modeling notations for Enterprise Architect, but can also provide validation rules, code round-trip engineering, model patterns etc. for specific domains and technologies.

Zachman Framework Technology

Sparx Systems' new MDG Technology underpins the organizational viewpoints and structures of the Zachman Framework with great depth, breadth and modeling integrity. Sparx Systems Enterprise Architect models are based on the UML 2.4 standard, with extensions for requirements modeling, testing, project management, mind mapping, and much more.

MDG Link

Model Driven Generation (MDG) Link add-ins provide application bridges between Enterprise Architect and popular third-party development tools including Visual Studio .NET, Eclipse, DOORS and Visio. This helps to achieve traceability across tools, efficient navigation from your model to code and synchronization of changes over time.

Visual Studio Link

The MDG Link for Visual Studio .NET provides a lightweight bridge between Enterprise Architect and Visual Studio .NET. Navigation between Enterprise Architect and Visual Studio .NET is enhanced with simple key combinations that take the user directly from an Enterprise Architect model to its corresponding code in Visual Studio .NET.

Learn More Buy Online Install

Eclipse Link

The MDG Link for Eclipse provides a lightweight bridge between Enterprise Architect and Eclipse. Navigation between Enterprise Architect and Eclipse is enhanced with simple key combinations that take the user directly from an Enterprise Architect model to its corresponding code in Eclipse.

DOORS Link

The MDG Link for DOORS provides a lightweight bridge for Sparx Systems' Enterprise Architect and Telelogic's DOORS. DOORS users can now incorporate requirements with the power of UML to fully integrate the software development process.

System Requirements

Native support for Microsoft® Windows 7, Windows Vista, Windows 2008, Windows 2003, Windows XP or Windows 2000. Mac/Linux users can install Enterprise Architect under WINE or CodeWeavers CrossOver.

Registering the Trial Version

The trial version cannot be registered - only the full version. Therefore when purchasing Enterprise Architect, you will need to download the full version and install it, then register it with your registration key. You will receive registration information at the time of purchase. This will include a username and password for the registered user area of our web site, a link to download the full version and your registration keys.

Transferring Projects from the Trial Version to a Registered Version

Please ensure that you have your .EAP files in a secure location when you to remove your trial version. To open the project with a registered version of Enterprise Architect, go to the File | Open menu and select your project from its new location.
